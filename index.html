The operating loop

For any product task, it runs the same five moves.

Not a menu dump — a disciplined sequence that ends in a decision you can defend.

1

Clarify only if ambiguous

Ask up to five sharp questions when the request is genuinely under-specified — otherwise proceed. No interrogation.

2

Pick the one right framework

Match the method to the decision — don't dump the whole toolbox on the table.

3

Output a decision-ready artifact

Recommendation first, reasoning second. An exec-ready answer — a scoring table, an eval plan, a PRD — not an essay.

4

Tag confidence

Every substantive recommendation carries a label — from VERY HIGH to SPECULATIVE — so you know how hard to lean on it.

5

Run the blind-spot check

Full ritual for high-stakes, irreversible calls; a one-line tag otherwise. Drop the ritual, never the thought.

Deep dive · Discovery

Build is cheap. Knowing what to build is the constraint.

Discovery is a weekly habit, not a project sprint (Torres). The trio — PM, design, eng — discovers together, so the people who build it hear the customer firsthand.

The Opportunity Solution Tree keeps everyone honest: every solution must ladder back to a real outcome. Split each solution into desirability / viability / feasibility / usability assumptions and test the riskiest first — plus Model-Capability for anything AI-backed.

JTBD job story — “When I [situation], I want to [motivation], so I can [outcome].” Frame the struggle, not the feature. The milkshake competes with bananas and boredom, not other shakes.

Deep dive · Metrics

Pick one metric. Ship its guardrail.

Every metric is a bet that it tracks real value. Name how it could rise while value falls — then pair it with a guardrail that must not degrade.

AARRR · Pirate Metrics — trace problems upstream
Acquisition
Activation
Retention
Revenue
Referral

A retention problem is often an Activation / time-to-value miss upstream — not an Acquisition one. Quick Ratio = (new + resurrected) ÷ churned; below 1, you leak faster than you fill.

North Star

One value-delivery metric + 3–5 movable inputs. A proxy, not the goal — re-validate quarterly that it still tracks retention/revenue. Marketplaces need two (supply + demand).

Vanity vs value

Signups, downloads, MAU are vanity. Activation “aha” within ~7 days and week-4 cohort retention are value. Goodhart: a North Star that becomes the target gets gamed.

Outcome, not event (AI)

When the product acts for the user, count jobs completed, not events fired. “Messages sent” rewards a chatbot that loops.

Deep dive · AI-Native PM

When building is cheap, proving it works becomes the job.

For any LLM-backed feature, the eval suite is the spec. Vibe-checks don't survive production. “Evals are the new PRD.”

Improve

Error analysis is the job

~60–80% of dev time is reading traces by hand: open-code the first thing that went wrong, cluster into a named failure taxonomy, build a metric only for the buckets that actually hurt.

Minimum Viable Quality (Nika)

Set a bar per feature in three tiers — do-not-ship · acceptable (ships with human-in-loop) · delight. Define a cost-per-inference envelope up front; review failure modes weekly.

Name the mode: generative vs agentic

Generative produces an artifact a human reviews → score the output. Agentic takes multi-step actions → add trajectory + tool-call evals. Misclassifying agentic as generative is the root of “the demo worked but it loops in prod.”

GTM · Positioning

Derive the five components in order.

1
Competitive alternatives

What they'd use if you didn't exist. “Do nothing” is usually your #1 competitor.

2
Unique attributes

Capabilities only you have versus those alternatives.

3
Value

Run the “so what?” test on each attribute until you reach value worth paying for.

4
Target market

The segment that cares most — best-fit, not all-addressable.

5
Market category

The frame of reference that makes the value obvious.

Substitutability test: swap your name for a competitor's. If the statement still reads true, it isn't differentiated.

Pricing

Price is a product decision — gate the build.

Have the willingness-to-pay conversation during design (Ramanujam). No WTP evidence, no spec. Capture ~10–25% of the value you quantify.

Feature triage
Leaders — few high-WTP differentiators you lead and charge for.
Fillers — modest-WTP nice-to-haves.
Killers — reduce WTP if charged. Never bundle a Leader with a Killer.

The blind-spot check

Match the ritual to the stakes.

High-stakes / irreversible — run all five
  1. Framework bias + the alternative you didn't pick
  2. Context bias (stage / size / industry assumed)
  3. Survivorship — are the examples outlier winners?
  4. Recency — durable, or trend-driven?
  5. A substantive contrarian view, not a token caveat
Routine / reversible — one line

Blind spot: [load-bearing assumption] · counter-take: [strongest opposing view]
